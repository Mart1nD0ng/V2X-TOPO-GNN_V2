# 基线对比实验方案（说人话版）

> 目标读者：你自己 + 审稿人。本方案设计一个**严谨、公平、能打**的基线对比实验，把"端到端可微
> 拓扑构造器"和一堆对手放在同一张擂台上比，并量化它离"最优"还有多远。
> 已有基础：`scripts/analysis/run_baseline_comparison.py`（已有：本方法 vs 通用 GraphSAGE vs 5 个启发式）、
> `src/v2x_env/baselines.py`（kNN / 贪心 / MST）、`run_advantage_map.py`（按密度×耦合扫的优势域）。
> 本方案是把这些**补齐成一张完整的基线表 + 一条"离最优有多近"的标尺**。

---

## 0. 一句话目标
证明三件事：
1. **学习 > 规则**：学出来的拓扑比手工规则和经典图算法都更可靠；
2. **架构/端到端 > 随便一个 GNN**：赢不只是因为"用了 GNN"，而是因为这套**端到端穿过共识评测器**的训练方式；
3. **离最优很近、但便宜得多**：用一次前向推理就逼近"贪心 oracle"的质量，而 oracle 要贵几个数量级。
并且额外证明一个对手做不到的事：**只有本方法能调节代价（延迟/能量）**，启发式只能停在一个代价点上。

---

## 1. 最重要的：公平原则（审稿人第一刀砍这里）
**所有方法只在"怎么给边打分/选边"这一步不同；其余一切都一样。**

- 同一个**构造器**：硬 top-k、同一个度预算（cap）；
- 同一个**解析评测器**：同样的 Avalanche 共识闭式（F/D/E），同样的物理（信道/干扰/重传）；
- 同一批**场景**：同样的车辆快照、同样的留出 seed；
- 同样的**信息**：每个方法看到的都是同一组候选边特征（距离/信道/成功率/SINR）。

→ 这样比出来的差异**只能归因于"拓扑决策本身"**，没有任何方法偷到额外信息或评测器优势。
（这正是项目里 F4.3 "fair heuristics" 已经在守的原则，本实验把它扩成完整的一桌。）

## 2. 擂台上的对手（5 组）

| 组 | 方法 | 说人话 | 来源 |
|---|---|---|---|
| **A 不规划（地板）** | flood 漫灌（保留所有候选边，度=候选上限） | "完全不规划会怎样" | 新增（最基本对照） |
| **B 手工规则** | nearest-k、best-channel-k、best-success-k、best-sinr-k、random-k | "一个明白人会手写的规则" | 已有 `run_baseline_comparison` |
| **C 经典图算法** | 度受限贪心、MST 骨干+增广、kNN | "网络设计教科书算法" | 已有 `baselines.py`，需接进来 |
| **D 学习型对手（关键）** | D1 通用 GNN(GraphSAGE)、D2 代理标签 GNN、D3 平均场训练 GNN | 见下，三个消融各砍一刀 | D1 已有，D2/D3 新增 |
| **E 上界 / oracle** | E1 逐节点贪心 oracle、E2(可选) MC-in-the-loop 搜索 | "理论上最好能多好" | 新增 |
| **★ 本方法** | 分层 GNN + 端到端耦合损失 | 待证明的主角 | 已有 |

**D 组三刀（这是论文最有说服力的部分，逐层剥离"为什么是我这套"）：**
- **D1 通用 GNN**（GraphSAGE，端到端同目标训练）：砍掉分层结构/结构偏置 → 回答"**架构值不值，还是随便个 GNN 就行**"。
- **D2 代理标签 GNN**：用一个**监督代理**训练（比如让 GNN 预测每条边的链路成功率），再 top-k 选边 → 回答
  "**端到端穿过共识评测器值不值，还是训练个'预测链路质量'的 GNN 就够了**"。
- **D3 平均场训练 GNN**（训练时用 Q=1 平均场评测器，评测时用 Q=21/MC）：回答
  "**淬火（quenched）货币是不是必须的**"——平均场看不见 query-spread 杠杆，规划器会退化成集中连边、吃亏。

**E 组上界（给"赢"一个标尺）：**
- **E1 逐节点贪心 oracle**：对每个节点，在评测器下贪心地选最能降低它失败率的那 k 条边。这是一个**强局部最优**，
  贵但可算，作为"边选择在这个预算下能达到的上界"。本方法越接近 E1，越说明"学到了接近最优的选择"。
- **E2（可选，少量小场景）MC-in-the-loop 搜索**：直接拿蒙特卡洛真值当目标去搜，金标准天花板，极贵，只在
  2–3 个小场景上跑，给一个"绝对天花板"参照。

## 3. 比什么指标（说人话）

| 指标 | 含义 | 怎么报 |
|---|---|---|
| **F**（失败率，越低越好） | 多少节点没达成正确共识 | 主指标；**Q=21 淬火闭式** + **蒙特卡洛真值复核**双货币 |
| **D / E**（延迟 / 能量） | 代价 | 两套子实验，见下 |
| **有效度** $1/\Sigma w^2$ | 每个节点实际用了几个邻居 | 显示本方法学到"负载最优的连边数" |
| **推理耗时 / 计算量** | 部署成本 | 本方法=一次前向；oracle=贵几个数量级 → "近 oracle 质量、启发式成本" |

**D/E 的两套子实验（关键区分）：**
1. **可靠性对齐**：所有方法都只优化 F（启发式天然只有一个 F 点）→ 在**相近代价**下比 F。
2. **代价可控性（只有学习型能做）**：本方法 + D 组学习型，用**完整耦合目标**在几个代价权重 w 上训练 →
   画出它们各自的 (F, D) 前沿；**启发式只能是一个点**（它没法权衡代价）。这一张图直接证明"耦合目标=我独有的能力"。

## 4. 实验协议（怎么跑才算数）

- **两个配置都跑**：① 重标定后的 `operating_point_v1`（代价有杠杆的难场景）；② `paper_environment_v1`
  （headline 地板场景）。两边都成立才稳。
- **样本 = seed × 场景**：学习型方法在 1 个训练场景上用 **5 个模型 seed** 各训一遍；**所有**方法再在
  **K 个留出场景**（不同 seed）上评测。分布 = 5×K，才有真正的置信区间。
- **统计**：每个方法报 **均值 ± 95% CI**；"本方法 CI 整条压在最好基线 CI 之下" = 统计显著分离；再加一个
  **配对检验**（同场景配对）报本方法 vs 最强基线的差值与 p 值。
- **真值校验**：把排名前几的方法再**穿一遍蒙特卡洛**，确认"闭式下的排名在真值下不翻车"（不是只在近似器里赢）。
- **跨域（不是一个点）**：在 `run_advantage_map` 的**密度×耦合网格**上重复 F 对比，给一张
  "本方法 vs 最强基线的优势热力图"，证明优势是**整片运行域**的，不是一个挑出来的点。

## 5. 每个对比"到底证明了什么"（一句话对应）

- vs **A 不规划** → 规划本身有用（最基本的 sanity）。
- vs **B 手工规则 / C 图算法** → 学习 > 人写规则和经典算法。
- vs **D1 通用 GNN** → 赢在**架构**，不是"用了 GNN"。
- vs **D2 代理标签 GNN** → 赢在**端到端穿过共识模型**，不是"预测链路质量"那种省事训练。
- vs **D3 平均场 GNN** → **淬火货币是必须的**（这条同时呼应项目的"货币诚实性"主线 F4.1）。
- vs **E1 贪心 oracle** → **逼近可达上界**，且成本低几个数量级。
- **代价可控性子图** → 只有本方法能把 (F,D,E) 沿前沿移动，启发式做不到。

---

## 6. 预期结果 + 诚实的风险（先把丑话说前面）
- **大概率赢且显著**：vs A/B/C（不规划、手工规则、图算法）——这是干净的 headline。
- **架构那一刀（vs D1 通用 GNN）可能只是"小赢"**：两个都是 GNN，差距可能不大，CI 可能擦边。**预案**：
  如实报"在难/耦合场景显著、在易场景打平"，并把 headline 放在 vs 启发式 + vs 平均场 + 代价可控性这三处。
- **D3 平均场那一刀大概率很显著**（项目已多次确认平均场盲于 query-spread）——这是强证据。
- **离 oracle 的差距（regret）要如实报**：本方法 = oracle 的多少倍 F、省了多少算力，两个数都给。
- 一句话原则：**哪里打平就如实写打平**（参照 F4.3 已有的诚实负面结果），不硬吹。

## 7. 代码：复用什么 + 新增什么
- **复用**：`run_baseline_comparison.py`（主框架、CI、公平构造器/评测器）、`baselines.py`（C 组）、
  `run_advantage_map.py`（跨域）、`validate_*_montecarlo.py`（E2/真值校验）、`run_pareto_frontier.train_one`（代价权重训练）。
- **新增（约 5 个小件）**：
  1. 把 **C 组图算法**（贪心/MST/kNN）接进 `run_baseline_comparison` 的方法表；
  2. **A 组 flood** 作为一个方法；
  3. **D2 代理标签 GNN**：一个"预测链路成功率"的监督训练分支；
  4. **D3 平均场 GNN**：把训练 `quenched_quadrature=1`，评测 Q=21/MC；
  5. **E1 逐节点贪心 oracle**：在评测器下逐节点贪心选边；
  6. **代价可控性子实验**：对学习型方法跑几个 w，画 (F,D) 前沿 + 启发式的单点。
- 产物：`result/baseline_comparison_v2/`：总表 JSON + 主图（F ±CI 柱状，按组上色）+ 代价前沿图 + 跨域优势热力图 + MC 校验表 + `RESULT.md`。
